**작지만 반복되는 불편함과 고민**에서 출발한 서비스입니다.  
상암 IT 센터에서 보내는 하루를 더 쾌적하고, 건강하고, 따뜻하게 만들기 위해

## 🔧 트러블슈팅

### 공통 

- 프로젝트 초기 단계에서 환경 변수 및 설정 파일 구조를 명확히 정하지 못해, 팀원마다 서로 다른 방식으로 환경 변수를 관리하고 있음을 중간에 확인하였다.
  - 문제의 핵심이 환경 변수 관리 방식의 불일치임을 파악한 후, 민감한 정보를 GitHub에 직접 포함하지 않고도 실행할 수 있도록 `.streamlit/secrets.toml` 기반의 구조로 프로젝트 설정을 통일 및 리팩토링하였다.
- 팀원 대부분이 GitHub 협업에 익숙하지 않아, repository 연결, 기능 브랜치 운영, main 브랜치 병합 과정에서 반복적인 혼선이 발생하였다.
  - GitHub 관련 이슈는 머지 충돌과 브랜치 관리 과정을 팀원들과 함께 해결하며, 각자의 시행착오와 해결 방법을 공유하였고, 주요 파일은 지속적으로 백업하여 문제 발생 시 즉시 복구 가능한 환경을 마련하였다.
